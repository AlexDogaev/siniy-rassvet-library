# Библиотека Синего Рассвета

Веб-версия библиотеки материалов канала. Хостится через GitHub Pages.

## Структура

- `index.html` — страница (трогать обычно не нужно)
- `data.json` — **все материалы**, редактируется здесь
- `logo.png` — логотип канала
- `README.md` — этот файл

## Как добавить новый материал

1. Открой `data.json` в репозитории и нажми карандашик («Edit»).
2. Добавь новый объект внутрь массива `items`. Скопируй любой существующий и поменяй поля.

Простая карточка со ссылкой на один документ:

```json
{
  "title": "Название материала",
  "type": "docx",
  "category": "Категория",
  "desc": "Короткое описание (1–2 предложения).",
  "url": "https://docs.yandex.ru/..."
}
```

Карточка-подборка с несколькими документами (раскрывается аккордеоном):

```json
{
  "title": "Название подборки",
  "type": "folder",
  "category": "Категория",
  "desc": "Описание подборки.",
  "children": [
    "https://docs.yandex.ru/...первый",
    "https://docs.yandex.ru/...второй"
  ]
}
```

3. Внизу страницы — **Commit changes** → коммит в `main`.
4. Через 30–90 секунд изменения появятся на сайте.

## Поля карточки

- `title` — заголовок (виден на карточке)
- `type` — иконка: `docx`, `xlsx` или `folder`
- `category` — категория, по ней работает фильтр (если категория новая — она автоматически появится в фильтрах)
- `desc` — короткое описание под заголовком
- `url` — прямая ссылка на документ (для одиночного материала)
- `children` — массив ссылок (для подборки из нескольких документов; поле `url` тогда не нужно)

## Как удалить материал

Удали соответствующий объект `{ ... }` из массива `items` (вместе с запятой). Закоммить.

## Если что-то сломалось

Сайт пустой или ругается на ошибку — почти всегда это сломанный JSON: лишняя или пропущенная запятая, незакрытая кавычка, лишний символ. Открой `data.json` и проверь, либо вставь содержимое в любой JSON-валидатор (например, jsonlint.com).

Можно откатить последнее изменение через историю коммитов в GitHub: открой `data.json` → **History** → найди нужную версию → **Revert**.

## Доступы

Для редактирования нужен GitHub-аккаунт и роль **Write** в этом репозитории. Приглашение делает владелец репо в Settings → Collaborators.
